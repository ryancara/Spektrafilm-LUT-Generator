# Spektrafilm State LUT Generator, OCIO CUBE build

This build removes the old hand-written CUBE path from the normal workflow.

## Current architecture

- **CLF** is generated natively from `gui_state.json` and is the reference output.
- **CUBE** is generated by first creating a temporary/reference CLF, then baking that CLF to a Resolve `.cube` using OpenColorIO's `ociobakelut`.
- The previous internal `make_cube_lut()` shaper/simple CUBE path is no longer used by the state wrapper.

## Requirements

For CLF export:

- A Python environment with Spektrafilm installed.

For CUBE export:

- A Python environment with Spektrafilm installed.
- OpenColorIO command-line tools available, especially `ociobakelut`.

On macOS with Homebrew:

```bash
brew install opencolorio
ociobakelut --help
```

## Recommended AP0 linear workflow

```bash
python spektrafilm_state_to_lut.py \
  --state gui_state.json \
  --format clf \
  --size medium \
  --input-mode aces-ap0 \
  --output-mode lut-default \
  -o "Generated LUTs/ap0_linear_reference.clf"
```

To create a Resolve CUBE from the same AP0 linear workflow:

```bash
python spektrafilm_state_to_lut.py \
  --state gui_state.json \
  --format cube \
  --cube-size 64 \
  --size medium \
  --input-mode aces-ap0 \
  --output-mode lut-default \
  -o "Generated LUTs/ap0_linear_ocio_cube64.cube"
```

Testing showed that OCIO-baked 64³ CUBE output matched the CLF much better than the old hand-written CUBE writer, and better than a 36³ CUBE in the shadows.

## Notes

- `--cube-size 64` is the current recommended default.
- `--keep-intermediate-clf` keeps the CLF that is normally generated temporarily when baking a CUBE.
- If `ociobakelut` is not on PATH, use `--ocio-bakelut /path/to/ociobakelut`.
